# Environment variables

Runtime configuration for the gateway application.

Environment variables configure how the gateway behaves at runtime. You set these in your ECS task definition or in `.env` files for local development.

## Core configuration

### ENVIRONMENT

Deployment environment identifier.

- **Required:** No
- **Default:** `dev`
- **Values:** `dev`, `test`

Determines which rate limit configuration file to load (`base.yaml` or `test.yaml`).

```bash
ENVIRONMENT=dev
```

### AWS_REGION

AWS region for API calls.

- **Required:** No
- **Default:** `us-east-1`

The region where you invoke Amazon Bedrock models and assume IAM roles.

```bash
AWS_REGION=us-east-1
```

### LOG_LEVEL

Application logging verbosity.

- **Required:** No
- **Default:** `INFO`
- **Values:** `DEBUG`, `INFO`, `WARNING`, `ERROR`, `CRITICAL`

Use `DEBUG` for development and `INFO` for deployed environments.

```bash
LOG_LEVEL=INFO
```

## OAuth and JWT

### OAUTH_JWKS_URL

JSON Web Key Set endpoint for token validation.

- **Required:** Yes (when authentication is enabled)
- **Default:** Empty (authentication disabled)

The gateway fetches public keys from this endpoint to verify JWT signatures.

```bash
# Auth0 example
OAUTH_JWKS_URL=https://<tenant>.us.auth0.com/.well-known/jwks.json

# Generic provider
OAUTH_JWKS_URL=https://your-provider.com/.well-known/jwks.json
```

### OAUTH_ISSUER

Expected JWT issuer claim.

- **Required:** Yes (if OAuth enabled)
- **Default:** Empty

Must match the `iss` claim in your JWT tokens.

```bash
OAUTH_ISSUER=https://<tenant>.us.auth0.com/
```

### JWT_AUDIENCE

Expected JWT audience claim.

- **Required:** No
- **Default:** `bedrockproxygateway`

Must match the `aud` claim in your JWT tokens.

```bash
JWT_AUDIENCE=bedrock-proxy-gateway-dev
```

### JWT_ALLOWED_SCOPES

Comma-separated list of allowed OAuth scopes.

- **Required:** No
- **Default:** `bedrockproxygateway:read,bedrockproxygateway:invoke,bedrockproxygateway:admin`

Tokens must contain at least one of these scopes.

```bash
JWT_ALLOWED_SCOPES=bedrockproxygateway:invoke,bedrockproxygateway:read
```

## Multi-account

### SHARED_ACCOUNT_IDS

Comma-separated AWS account IDs with Amazon Bedrock access.

- **Required:** Yes
- **Default:** Empty

The gateway assumes IAM roles in these accounts to invoke models.

```bash
SHARED_ACCOUNT_IDS=123456789012,234567890123,345678901234
```

### SHARED_ROLE_NAME

IAM role name in shared accounts.

- **Required:** No
- **Default:** `shared-account-role`

The gateway assumes this role in each shared account. Must be identical across all accounts.

```bash
SHARED_ROLE_NAME=BedrockGatewayRole
```

### STS_ROLE_SESSION_NAME_SUFFIX

Random suffix for STS session names.

- **Required:** Yes (when using multi-account routing)
- **Default:** Empty

Generated by Terraform. Validates role assumptions are from this gateway.

```bash
STS_ROLE_SESSION_NAME_SUFFIX=a1b2c3d4e5f6g7h8
```

## AWS service endpoints

### BEDROCK_RUNTIME_VPC_ENDPOINT_DNS

VPC endpoint DNS for Amazon Bedrock Runtime.

- **Required:** Yes (when using VPC endpoints)
- **Default:** Empty (uses public endpoint)

Routes Bedrock API calls through your VPC endpoint.

```bash
BEDROCK_RUNTIME_VPC_ENDPOINT_DNS=vpce-xxx.bedrock-runtime.us-east-1.vpce.amazonaws.com
```

### STS_VPC_ENDPOINT_DNS

VPC endpoint DNS for AWS STS.

- **Required:** Yes (when using VPC endpoints)
- **Default:** Empty (uses public endpoint)

Routes STS calls through your VPC endpoint.

```bash
STS_VPC_ENDPOINT_DNS=vpce-yyy.sts.us-east-1.vpce.amazonaws.com
```

## Rate limiting

### VALKEY_URL

Valkey (Redis) connection URL.

- **Required:** Yes (if rate limiting enabled)
- **Default:** `redis://localhost:6379`

The gateway connects to Valkey to store rate limit state.

```bash
# Production (SSL enabled automatically)
VALKEY_URL=rediss://valkey.abc123.serverless.use1.cache.amazonaws.com:6379

# Local development
VALKEY_URL=redis://localhost:6379
```

### ELASTICACHE_USE_IAM

Enable IAM authentication for ElastiCache.

- **Required:** No
- **Default:** `false`

When enabled, uses IAM authentication instead of password-based auth. Requires `ELASTICACHE_CLUSTER_NAME` and `ELASTICACHE_USERNAME`.

```bash
ELASTICACHE_USE_IAM=true
```

### ELASTICACHE_CLUSTER_NAME

ElastiCache cluster name for IAM authentication.

- **Required:** Yes (if `ELASTICACHE_USE_IAM=true`)
- **Default:** None

```bash
ELASTICACHE_CLUSTER_NAME=vlk-myapp-dev
```

### ELASTICACHE_USERNAME

ElastiCache username for IAM authentication.

- **Required:** Yes (if `ELASTICACHE_USE_IAM=true`)
- **Default:** `default`

```bash
ELASTICACHE_USERNAME=default
```

### RATE_LIMITING_ENABLED

Enable rate limiting.

- **Required:** No
- **Default:** `false` for dev, `true` for test

When disabled, all requests bypass rate limit checks.

```bash
RATE_LIMITING_ENABLED=true
```

## Observability

### OTEL_SERVICE_NAME

Service name for OpenTelemetry.

- **Required:** No
- **Default:** `bedrock-proxy-gateway`

Identifies the service in traces and metrics.

```bash
OTEL_SERVICE_NAME=bedrock-proxy-gateway-dev
```

### OTEL_EXPORTER_OTLP_ENDPOINT

OpenTelemetry collector endpoint.

- **Required:** No
- **Default:** `http://localhost:4317`

The gateway sends traces and metrics to this endpoint.

```bash
OTEL_EXPORTER_OTLP_ENDPOINT=http://localhost:4317
```

### OTEL_SDK_DISABLED

Disable OpenTelemetry instrumentation.

- **Required:** No
- **Default:** `false`

Set to `true` for local development to reduce overhead.

```bash
OTEL_SDK_DISABLED=false
```

## Set environment variables

### In ECS (Terraform)

Environment variables are set in the ECS task definition:

```hcl
resource "aws_ecs_task_definition" "api_task" {
  container_definitions = jsonencode([
    {
      name = "bedrock-proxy-gateway"
      environment = [
        {
          name  = "ENVIRONMENT"
          value = var.environment
        },
        {
          name  = "OAUTH_JWKS_URL"
          value = var.oauth_jwks_url
        }
      ]
    }
  ])
}
```

### For local development

Create a `.env` file:

```bash
# .env
ENVIRONMENT=local
AWS_REGION=us-east-1
LOG_LEVEL=DEBUG
```

Load with Docker Compose:

```yaml
services:
  gateway:
    env_file:
      - .env
```

### Using AWS Secrets Manager

For sensitive values, use Secrets Manager:

```bash
# Store secret
aws secretsmanager create-secret \
  --name bedrock-proxy-gateway/dev/oauth \
  --secret-string '{"client_id":"xxx","client_secret":"yyy"}'
```

Reference in ECS task definition:

```json
{
  "secrets": [
    {
      "name": "OAUTH_CLIENT_SECRET",
      "valueFrom": "arn:aws:secretsmanager:region:account:secret:bedrock-proxy-gateway/dev/oauth:client_secret::"
    }
  ]
}
```

## Troubleshooting

### Variable not loading

**Check:**

- Variable name spelled correctly (case-sensitive)
- Variable set in ECS task definition
- Container restarted after changes

```bash
# View task definition
aws ecs describe-task-definition --task-definition bedrock-proxy-gateway-dev
```

### OAuth validation fails

**Check:**

- `OAUTH_JWKS_URL` accessible from ECS
- `OAUTH_ISSUER` matches token issuer
- `JWT_AUDIENCE` matches token audience

```bash
# Decode token to check claims
echo $TOKEN | cut -d. -f2 | base64 -d | jq
```

### Valkey connection fails

**Check:**

- `VALKEY_URL` format correct
- Security group allows port 6379
- Valkey endpoint accessible from ECS

For more help, refer to [TROUBLESHOOTING.md](../TROUBLESHOOTING.md).
